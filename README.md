# ESP32 기반 스마트 글러브 (Unity 연동 시스템)

ESP32 DevKitC V4 개발 보드와 MPU6050 IMU 센서, 그리고 5개의 가변저항을 활용하여 사용자의 손가락 움직임과 손목의 회전 데이터를 실시간으로 측정하고 Unity로 송신하는 시스템

---

# 시스템 사양 및 통신 규격
- 주요 부품: ESP32 DevKitC V4, MPU6050 (6축 가속도/자이로 센서), 가변저항 5개
- 통신 방식: USB Serial (가상 COM 포트)
- 보레이트 (Baud Rate, 통신 속도): 115200 bps
- 데이터 갱신 주기: 약 60Hz (delay 16ms)
- 출력 포맷 (CSV 형태의 단일 라인 문자열):
  ```text
  c0,c1,c2,c3,c4,qw,qx,qy,qz\n

---

# 알고리즘

## 1. 2초 자동 영점 조절
   IMU 센서는 초기 부팅 시를 기준으로 절대 각도를 연산하므로, 사용자가 글러브를 착용한 기본 자세를 기준점으로 재설정할 필요가 있다.
   ### 동작방식
   시스템 리셋 직후 2000ms 동안 MPU6050 내부 DMP FIFO 버퍼로부터 실시간 데이터를 누적 수집한다.
   ### 수식
   수집된 샘플 개수(N) 동안 측정한 qw, qx, qy, qz의 평균을 구한 뒤에 크기를 1로 마주는 nomalization을 거쳐 $$q_{base}$$ 를 정의한다.

   ## 2. 상대 회전 연산
   초기 영점 조절이 완료되면, 센서가 물리적으로 회전하더라도 유니티 공간에서는 초기 기준점에 상대적으로 회전한 양만 표현해야한다.
   ### 동작방식
   실시간으로 측정되는 현재 q 값에 초기 기준 q_base의 역행렬을 곱하여, 초기 오프셋이 제거된 순수 상대 회전량인 $$q_{re}$$을 계산할 수 있다.
   ### 수식
   $$q_{re} = q_{base} \otimes q_{current}$$
   이 연산을 통해서 시스템이 시작 되었을 때 어떤 방향을 바라보고 있었든 상관없이 리셋 직후의 상태를 항상  $(w=1, x=0, y=0, z=0)$, 즉 회전각 0도로 고정한다.

   ## 3. ADC 데이터 선형 정규화
   ESP32 의 ADC는 기본적으로 12-bit를 사용하고, 이를 통해서 가변저항의 전압 신호를 처리한다.
   ### 동작방식
   손가락을 완전히 폈을 때의 아날로그 최솟값과 완전히 굽혔을 때의 최대값을 매핑하여 0.0(핌) - 1.0(구부림)의 실수 범위로 선형 정규화한다.
   ###수식
   $$\text{Normalized Value} = \frac{\text{Raw ADC} - minRaw}{maxRaw - minRaw}$$
